# Hello, World

Every programmer starts here. And honestly, there's no better place to begin a blog about systems programming than with the most fundamental program ever written.

## Why Systems Programming?

I've always been fascinated by what happens *beneath* the code. When you write `printf("Hello, World\n")`, what actually happens? How does that string make it from your source code to the screen?

Most developers never need to think about this. But I'm not most developers.

## What You'll Find Here

This blog is a journal of my journey into the depths of computing:

- **C programming** — memory management, data structures, and fighting the compiler
- **Systems concepts** — how operating systems, compilers, and hardware actually work
- **Project write-ups** — detailed breakdowns of things I've built from scratch
- **Security research** — binary exploitation, reverse engineering, and vulnerability analysis

## The Philosophy

I believe the best way to understand something is to build it yourself. Not with frameworks. Not with libraries. From scratch, with nothing but a compiler and determination.

Welcome to Muffin Man Labs. Let's build something.
